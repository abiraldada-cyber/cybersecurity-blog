NETWORKING BASELINE

Computer Networking

How data moves across networks and how we audit it

1. Basic: How Systems Communicate

Before you can secure a network, you must understand how data moves. When you click a link on your browser, your computer doesn't just instantly show the page. It slices the request into thousands of small envelopes called **packets** and sends them across the wire.

To understand this systematically, engineers use the **OSI Model** (conceptual 7 layers) and the **TCP/IP Model** (practical 4 layers). Data gets wrapped (encapsulated) in headers at each layer as it travels down, and unwrapped (decapsulated) as it travels up on the receiving machine.

OSI Layer Layer Name TCP/IP Layer Role in the Journey Security Concerns
L7 Application Application User interface & application data (HTTP, DNS) Payload exploits (SQLi, XSS)
L6 Presentation Data formatting, compression, encryption (SSL/TLS) Certificate spoofing
L5 Session Manages communication sessions between endpoints Session hijacking
L4 Transport Transport Port-to-port transmission & reliability (TCP, UDP) SYN floods, port scanning
L3 Network Internet Routing packets across networks via IP addresses IP spoofing, DDoS routing
L2 Data Link Network Access MAC address communication on the local switch ARP poisoning, MAC flooding
L1 Physical Physical media, cables, and electrical signals Physical wiretapping

2. Practical: Packet Analysis with Wireshark

Studying models is great, but seeing real traffic makes it click. **Wireshark** is a tool that captures raw network packets and allows you to inspect their contents.

Inspecting a Plaintext HTTP Request

If a website does not use encryption (HTTP instead of HTTPS), anyone on the same local network can capture your packets and read your credentials.

In the terminal mockup to the right, we show what a captured HTTP packet looks like in Wireshark. Since it is plaintext, the server request and authorization headers are fully readable, exposing usernames and passwords.

Wireshark Packet Capture - TCP Stream

GET /login.php HTTP/1.1

Host: vulnerable-blog.com

User-Agent: Mozilla/5.0 (Kali Linux)

Accept: text/html,application/xhtml+xml

Authorization: Basic YWJpcmFsOjEyMzQ1Ng==

Note: "YWJpcmFsOjEyMzQ1Ng==" is Base64 encoded.
Decoding it reveals: abiral:123456

Packet Sniffing

Wireshark hooks directly into your network card, putting it in "promiscuous mode" to capture all frames passing through the wire.

Filters

Use filters like http, dns, or ip.addr == 192.168.1.10 to isolate target streams out of thousands of packets.

TCP Handshake Analysis

Inspect the three-way handshake flags (SYN ➔ SYN-ACK ➔ ACK) to trace connection attempts and identify SYN flood attacks.

3. Advanced: Secure Infrastructure

To counter packet sniffing and routing spoofing, we deploy cryptographically secured protocols at the infrastructure level.

By moving away from plaintext protocols and enforcing secure communication baselines, we protect our data in transit.

HTTPS & TLS 1.3

Encrypts application-layer data (HTTP) before passing it down. Even if an attacker captures the packets using Wireshark, the payload appears as unreadable ciphertext.

DNSSEC

Adds cryptographic signatures to DNS records. This prevents attackers from poisoning your local DNS cache and redirecting you to phishing servers.

VPN Tunneling

Encapsulates your entire network packet (including the headers) inside a secure IPsec or WireGuard tunnel, hiding both your data and your source IP address.
