Mastering Kali Linux CLI

The primary toolkit for security auditing, penetration testing, and digital forensics.

Where the Journey Begins

Hey there! Welcome to the start of my learning log. If you are new to cybersecurity, this is where we begin. Before we can analyze complex web attacks or dissect malware, we must learn how to navigate the command line.

Linux isn't just an operating system; it's a security professional's primary interface. Kali Linux comes pre-loaded with hundreds of open-source utilities for auditing networks, crack-testing passwords, and exploring software configurations. But to use those tools effectively, you must master the shell.

Let's start by learning these 20 fundamental commands. We will progress from basic file navigation to data parsing, and finally to advanced network auditing utilities.

20 Essential Linux / Kali Commands

Structured from simple navigation to advanced security assessments

1. Basic Navigation & Operations (Level: Beginner)

Command Simple Explanation Cybersecurity Use-Case
pwd "Print Working Directory". Shows the exact path of the folder you are currently inside. Helps you confirm your path position before running automated scripts or dumping output files.
ls "List". Displays all the files and folders inside the directory you are looking at. Identify hidden script files (like .bash_history) using the ls -la flag.
cd "Change Directory". Moves you from one folder to another. Navigate to configuration folders (e.g. cd /etc/nginx/) to check configuration permissions.
cat "Concatenate". Reads the text inside a file and prints it directly on the terminal screen. Read local configuration files, hashes, or system text records (e.g. cat /etc/resolv.conf).
mkdir "Make Directory". Creates a brand-new empty folder. Organize your workspace when starting a security audit (e.g. mkdir nmap_results).
cp "Copy". Duplicates a file and places the copy in a location you choose. Create a backup copy of a configuration file before modifying it (e.g. cp config.conf config.conf.bak).

2. System Controls & Manipulation (Level: Intermediate)

Command Simple Explanation Cybersecurity Use-Case
grep Searches through text files for specific words or patterns and highlights them. Parse system logs to find unauthorized activities (e.g. grep "Failed password" /var/log/auth.log).
chmod "Change Mode". Updates access permissions (read, write, execute) of a file or directory. Restrict private keys so only the owner can read them (e.g., chmod 600 id_rsa).
sudo "Superuser Do". Runs a command with administrator/root-level permissions. Install auditing packages or run raw socket packets (e.g. sudo nmap -sS target).
apt "Advanced Package Tool". The app manager for Debian systems, used to install, update, and remove software. Install security auditing packages directly on Kali Linux (e.g., sudo apt install subfinder -y).
systemctl Manages system services (like starting, stopping, or restarting background services). Start a local SSH server or web server to host exploitation payload files (e.g. sudo systemctl start ssh).
ps "Process Status". Lists running processes on the system. Review active processes on a compromised computer to detect unauthorized shells (e.g., ps -aux).
df "Disk Free". Reports how much disk space is available on the system. Ensure you have enough space before downloading large forensic images or traffic captures.

3. Network Auditing & Utilities (Level: Advanced)

Command Simple Explanation Cybersecurity Use-Case
ping Sends packets to an IP address to check if a remote host is active and responsive. Perform basic network connectivity checks (e.g. ping -c 4 192.168.1.1).
curl Transfers data to or from a server. Used to interact with APIs and download files from the web. Inspect HTTP headers to check server software versions (e.g. curl -I http://target.com).
ss "Socket Statistics". Lists active network connections and ports open on your system. Audit your system's open ports and locate running services (e.g., ss -tulpn).
nmap "Network Mapper". Scans remote servers to discover open ports, services, and operating system details. Find vulnerable entry points during security assessments (e.g. nmap -sV -T4 10.10.10.5).
netcat / nc Known as the "TCP/IP Swiss Army knife". Used to read and write data across network connections. Test raw socket connections or listen for incoming test connections (e.g., nc -lvnp 4444).
tcpdump A command-line packet sniffer. It captures and displays network traffic passing through your interface. Monitor network traffic to analyze unencrypted login attempts (e.g. sudo tcpdump -i eth0 port 80).
find Searches the directory tree for files matching specific criteria (like name, size, or permissions). Locate misconfigured SUID binaries that could allow privilege escalation (e.g. find / -perm -4000 -type f 2>/dev/null).
