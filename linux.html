Linux System Architecture

Understanding filesystem hierarchies, user permissions, and process management

1. Basic: Filesystem and Directory Hierarchy

In Windows, files are stored on separate drives (C:\, D:\). Linux takes a different approach: **everything is a file**, organized under a single root directory represented by a forward slash (/).

When auditing a Linux system, security analysts look at specific directories to verify configurations or audit logs:

/etc

Contains all system configuration files. Auditing files like /etc/passwd (list of users) is a common security step.

/var/log

Stores system, authentication, and service logs (like Apache or Nginx logs) used to detect suspicious access.

/tmp

A temporary directory that any user can write to. Attackers often download and run scripts here.

2. Practical: User Permissions and Octal Values

Linux is a multi-user operating system. Access is managed through strict permissions divided into three groups: **Owner** (User), **Group**, and **Others** (everyone else).

Decoding Permissions

Permissions are represented symbolically by three flags: Read (r), Write (w), and Execute (x).

We assign numeric values to these flags to change permissions using the chmod command:
r = 4 | w = 2 | x = 1 | - = 0

chmod 755 script.sh - Owner has full access (7). Group and Others can read and run it (5), but cannot edit it.
chmod 600 id_rsa - Owner can read and edit (6) their SSH private key. Everyone else is fully blocked (000).
Permissions breakdown - ls -l output

-rwxr-xr-x 1 abiral users 420 Jun 14 06:30 script.sh

Flag Breakdown:

  • - : Standard file indicator
  • rwx : Owner permissions (4 + 2 + 1 = 7)
  • r-x : Group permissions (4 + 0 + 1 = 5)
  • r-x : Others permissions (4 + 0 + 1 = 5)

3. Advanced: Process Hierarchy & SUID Auditing

Mastering system administration requires understanding how processes run and identifying privilege risks.

By monitoring active processes and auditing permission states, we prevent privilege escalation.

Process Trees & Sockets

Every program running on Linux starts as a child process of a parent process.

Using command filters like ps aux --forest, analysts trace parent-child process connections to find hidden backdoor shells. We also audit system ports using ss -tulpn to see which processes are opening active network sockets.

SUID Privilege Risks

The SUID (Set Owner User ID) is a special permission bit assigned to files.

When a file has the SUID bit set, it runs with the privileges of the file's owner (often root) rather than the user running it. If a SUID binary is misconfigured, attackers can exploit it to escalate their permissions to root.
